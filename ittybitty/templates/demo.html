itty-bitty sea of glass

the URL is the document

This entire page is stored in the URL. There is no server, no database, no API. The HTML is compressed, encoded into the URL fragment, and decoded client-side. Share the link and the page travels with it.

how it works

seaof.glass/ittybitty/#/Page-Title/hg0,H4sIAAAA...compressed data...

The fragment after # contains everything: an optional title, a 3-character header (type + format + encryption flag), and the gzip-compressed, base64url-encoded payload.

features

code editor

CodeMirror 6 with syntax highlighting, line numbers, and live preview as you type.

encryption

AES-256-GCM with PBKDF2 key derivation. Passphrase never leaves your browser.

zero server

Pure static site. Content lives in the URL fragment — never sent to any server.

offline ready

Service Worker caches the app. Create and view pages without a network connection.

multi-format

HTML, Markdown, SVG, JSON, plain text. Auto-detected or explicitly set.

shareable

QR code generation, Web Share API, copy to clipboard. One link, everything included.

url format

Type:    h(tml)  m(arkdown)  s(vg)  j(son)  t(ext)
Format:  g(zip)  n(one)  l(zma, legacy decode only)
Encrypt: 0 = plaintext, 1 = AES-256-GCM encrypted

Example: #/My-Page/hg0,H4sIAAAA...
         #/Secret/hg1,salt+iv+ciphertext...

encryption details

When encryption is enabled, the payload is first gzip-compressed, then encrypted with AES-256-GCM. The key is derived from your passphrase using PBKDF2 with 100,000 iterations of SHA-256 and a random 16-byte salt. The salt and 12-byte IV are prepended to the ciphertext. Recipients enter the passphrase to decrypt — the key is never transmitted anywhere.

legacy compatibility

URLs from the original itty-bitty project are automatically detected and decoded. LZMA decompression is supported for reading legacy links.

This page you're reading right now is ~3KB of HTML, compressed to ~1.5KB, living entirely inside the URL in your address bar. No server involved.
